How Escalation Works
Escalation Timeline
  1. Message Received: Support request comes in via Telegram
  2. Primary Notification: On-call person is notified immediately
  3. Timeout Check: System waits for the configured timeout period
  4. Escalation: If no response, backup staff are notified
  5. Repeat: Process continues up to max escalation level
Priority Levels
  • Critical Immediate attention required (outages, security)
  • High Important issues affecting multiple users
  • Normal Standard support requests
  • Low General questions and minor issues
